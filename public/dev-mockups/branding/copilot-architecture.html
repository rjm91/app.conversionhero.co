Concept · Architecture sketch

Voice → Action

An in-app AI Copilot that can read what's on the page and take typed actions on it — so you can speak a change and watch it happen. Designed to scale to any page, not just the Agreement Builder.

Not built — this is a thinking doc

The copilot is just more React + Next.js — no new runtime, no rip-and-replace.

2 The two primitives

👁️ Readable state
Each page tells the agent what's currently on screen. So "rename it" is a one-field change, not a full rebuild. Fixes today's "blind agent" problem.
useAgentState('agreement', form)
⚡ Actions
Each page registers things the agent can do — name, description, typed params, and a handler that updates the page live.
registerAgentAction({
  name: 'updateAgreement',
  params: { customScope, monthly, term },
  run: (p) => setForm(...)
})

6 Roll-out plan

  • 1
    Build the registry + convert the Agreement Builder to stateful, live edits (no accept gate). Fixes today's reliability and proves the pattern.
  • 2
    Add the Leads page as provider #2 — update status, schedule appointment by voice. ~A small add, not a rebuild.
  • 3
    Scripts, Dashboard, Payments… each one is "register state + actions." The copilot spreads across the app.

7 Prior art (the names & standards)

Tool use
Anthropic's function-calling — what you already use. The core primitive.
MCP
Model Context Protocol — Anthropic's open standard for exposing tools + context uniformly. The "make it pluggable" north star.
CopilotKit
Independent open-source React framework (not Microsoft) — packages this exact "readable state + actions" idea. Adopt it, or build the lightweight in-house version shown here.
The pattern
Industry name: an "AI Copilot" / "agent actions" layer. Your product name "Voice → Action" works great on top.
